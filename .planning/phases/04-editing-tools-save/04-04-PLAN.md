# Plan 04-04: NIfTI Save API & Workflow

<objective>
Enable the user to save drawing modifications resulting in a valid NIfTI header file.
</objective>

<requirements>
- SAVE-01: Save segmentation back to disk (to companion file next to volume)
- SAVE-02: Allow save as specific filename
- SAVE-03: Ensure saved file is valid NIfTI with correct header/origin
</requirements>

## Tasks

<task>
<action>
Add `POST /api/volumes/{volume_id}/segmentations` in `server/api/segmentations.py`.
- Import `Request` from fastapi.
- Endpoint signature: `async def save_segmentation(volume_id: str, request: Request, filename: str)`.
- Re-use `_catalog` from `server.main` to find the volume by `volume_id` to get `volume_meta.path`. If not found, raise 404.
- Read binary data from python request: `body = await request.body()`.
- Import `nibabel` and `numpy`.
- Load source volume: `ref_img = nib.load(volume_meta.path)`. Canonicalize it: `canonical_ref = nib.as_closest_canonical(ref_img)`.
- Use the shape `dimX, dimY, dimZ = canonical_ref.shape`. Reshape the client binary input `np.frombuffer(body, dtype=np.uint8)` into shape `(dimZ, dimY, dimX)` and then `.transpose(2, 1, 0)` so it's shape `(dimX, dimY, dimZ)` back in NIfTI orientation.
- Create new NIfTI image: `new_img = nib.Nifti1Image(transposed_array, canonical_ref.affine, canonical_ref.header.copy())`.
- Call `new_img.set_data_dtype(np.uint8)`.
- Save it to disk: `nib.save(new_img, Path(volume_meta.path).parent / filename)`.
- Clear or append to the cache in `_segmentation_catalog` safely or just return HTTP 200.
</action>
