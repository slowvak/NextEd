<task>
<action>
Implement Label Visibility in canvas rendering by modifying `client/src/viewer/overlayBlender.js`.
- During `.notify()` updates or inside `overlayBlender.js` directly, we must ensure invisible labels are not rendered.
- Replace `buildColorLUT(labels)` in `ViewerState.js` or `overlayBlender.js` so it produces an alpha of `0` in the LUT for labels where `isVisible === false`.
- Specifically, in `buildColorLUT`:
  ```javascript
  const isVisible = label.isVisible === undefined || label.isVisible;
  lut[idx] = isVisible ? label.color.r : 0;
  lut[idx + 1] = isVisible ? label.color.g : 0;
  lut[idx + 2] = isVisible ? label.color.b : 0;
  // Use idx+3 for alpha signaling if overlayBlender needs it?
  ```
  Since `overlayBlender` currently checks `if (r===0 && g===0 && b===0)`, setting them to 0 effectively hides the label. Ensure `setup` correctly writes 0s.
</action>
