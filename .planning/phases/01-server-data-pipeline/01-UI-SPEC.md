# Phase 1 — UI Design Contract

> Visual and interaction contract for the volume browser UI. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (vanilla JS + DOM) |
| Icon library | none (text labels and CSS shapes only in Phase 1) |
| Font | system-ui, -apple-system, "Segoe UI", Roboto, sans-serif |

**Rationale:** Vanilla JS + Canvas stack per RESEARCH.md. No React, no shadcn. System font stack avoids network requests and matches the utilitarian, tool-focused aesthetic of ITK-SNAP.

## Typography

| Role | Size | Weight | Line Height | Usage in Phase 1 |
|------|------|--------|-------------|-------------------|
| Body | 14px | 400 (regular) | 1.5 | Metadata values, list item filenames |
| Label | 12px | 600 (semibold) | 1.4 | Metadata field labels ("Study Description:", "File Date:"), format badges |
| Heading | 18px | 600 (semibold) | 1.2 | Panel titles ("Volumes", "Volume Details") |

Three sizes only. 14px body matches the dense, information-rich aesthetic of medical imaging tools (ITK-SNAP uses small type for its inspector panels). 12px labels are legible at the secondary role. 18px headings provide hierarchy without wasting vertical space.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | #1e1e1e | Page background, main content area |
| Secondary (30%) | #2d2d2d | Sidebar panel background, detail panel background, list item hover |
| Accent (10%) | #4a9eff | "Open Volume" button background, selected volume highlight border-left, active state indicator |
| Destructive | not applicable | No destructive actions in Phase 1 |
| Text primary | #e0e0e0 | Body text, filenames, metadata values |
| Text secondary | #a0a0a0 | Metadata labels, secondary info (dimensions, spacing) |
| Text on accent | #ffffff | Text inside accent-colored buttons |
| Border | #3a3a3a | Panel borders, list item separators |
| Surface hover | #363636 | Volume list item hover state |
| Badge DICOM | #2a5a3a (bg) / #6fcf97 (text) | DICOM format badge |
| Badge NIfTI | #3a4a5a (bg) / #6fb3cf (text) | NIfTI format badge |

Accent reserved for: "Open Volume" button, selected volume left-border highlight, loading spinner/progress. Never applied to badges, metadata text, or panel backgrounds.

**Dark theme rationale:** ITK-SNAP reference uses dark backgrounds for all panels. Medical imaging tools universally use dark themes because bright backgrounds cause eye fatigue during extended viewing sessions and interfere with perception of grayscale image data. Phase 2+ viewer panels will use #000000 for image backgrounds.

### Volume List Panel (left sidebar)
- Width: 280px fixed
- Background: #2d2d2d
- Border-right: 1px solid #3a3a3a
- Panel heading "Volumes" at 18px semibold, padding 16px
- List is scrollable (overflow-y: auto), fills remaining height below heading
- Each list item:
  - Padding: 8px 16px
  - Border-bottom: 1px solid #3a3a3a
  - Filename at 14px regular #e0e0e0 (truncate with ellipsis if > 240px)
  - Format badge inline-end: "DICOM" or "NIfTI" at 12px semibold, pill-shaped (border-radius 4px, padding 2px 8px)
  - Dimensions below filename: "{X} x {Y} x {Z}" at 12px regular #a0a0a0
  - Hover: background #363636
  - Selected: background #363636, left border 3px solid #4a9eff, padding-left adjusted to 13px to compensate

### Detail Panel (right content area)
- Fills remaining width
- Background: #1e1e1e
- Padding: 24px
- Panel heading "Volume Details" at 18px semibold #e0e0e0

#### Metadata Display (when a volume is selected)
- Metadata rendered as a definition-list style layout:
  - Label at 12px semibold #a0a0a0, margin-bottom 4px
  - Value at 14px regular #e0e0e0, margin-bottom 16px
- Fields displayed for ALL volumes:
  - Filename
  - Format (DICOM / NIfTI)
  - Dimensions (X x Y x Z)
  - Voxel Spacing (sx x sy x sz mm)
  - Modality (CT / MR / Unknown)
- Additional fields for DICOM volumes:
  - Study Description
  - Series Description
  - Number of Files
- Additional fields for NIfTI volumes:
  - File Date (formatted as YYYY-MM-DD)
- "Open Volume" button:
  - Margin-top: 24px
  - Background: #4a9eff
  - Color: #ffffff
  - Font: 14px semibold
  - Padding: 8px 24px
  - Border-radius: 6px
  - Hover: background #3a8aee
  - Active: background #2a7ade
  - Cursor: pointer

#### Loading State (while volume data is transferring)
- Button text changes to "Loading..." with opacity 0.7
- Button is disabled (pointer-events: none)
- Optional: small inline CSS spinner (8px) left of button text

#### Success State (volume loaded)
- Button text changes to "Loaded" with background #2a5a3a and text #6fcf97
- After 2 seconds, button reverts to "Open Volume"
- Console logs: "Volume loaded: {filename}, ArrayBuffer size: {bytes}"

## Interaction Contract

### Volume Selection
- **Trigger:** Click on volume list item
- **Response:** Immediately highlight selected item (left border accent), populate detail panel with metadata from the already-fetched volume list data (no additional API call needed -- metadata comes from GET /api/volumes)
- **Latency target:** <50ms (DOM manipulation only, data already in memory)

### Open Volume
- **Trigger:** Click "Open Volume" button
- **Response:** Fetch binary data from GET /api/volumes/{id}/data, show loading state on button
- **Success:** Log ArrayBuffer receipt to console, show success state on button
- **Error:** Show error state on button ("Failed to load"), log error to console
- **Latency target:** Depends on volume size. Typical NIfTI brain MR (~50MB) should transfer in <2s on localhost.

### Keyboard Navigation
- None required in Phase 1. Keyboard shortcuts begin in Phase 4 (KEYS-01).

## Accessibility Baseline

- All interactive elements must be keyboard-focusable (tab order: volume list items, then Open Volume button)
- Volume list items use `role="listbox"` / `role="option"` with `aria-selected`
- Open Volume button uses semantic `<button>` element
- Color contrast: #e0e0e0 on #1e1e1e = 11.3:1 (exceeds WCAG AAA); #a0a0a0 on #1e1e1e = 5.6:1 (exceeds WCAG AA)
- Focus ring: 2px solid #4a9eff with 2px offset on all interactive elements

## Phase-Forward Notes
