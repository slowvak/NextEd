<objective>
Add RAS+ canonical orientation normalization and auto-windowing metadata to the server's volume loading pipeline.

Purpose: The client viewer needs volumes in a consistent RAS+ axis order so it can use fixed axis mappings (axis 0 = R-L, axis 1 = A-P, axis 2 = I-S) without interpreting affine matrices. Auto-windowing metadata (5th-95th percentile) lets the client display reasonable contrast on first open (WLVL-01). Voxel spacing in RAS+ order is required for anisotropic display correction (VIEW-06).

Output: Updated server loaders that normalize volumes to RAS+ before serving, extended volume metadata with voxel_spacing/window_center/window_width fields, and tests proving correctness.
</objective>

  <action>
Modify the NIfTI loader (`load_nifti_volume` in server/loaders/nifti_loader.py):
1. After loading with `nib.load()`, apply `nib.as_closest_canonical(img)` to reorient to RAS+
2. Extract data with `np.ascontiguousarray(canonical.get_fdata(dtype=np.float32))` — C-contiguous is critical for correct client-side array indexing
3. Get voxel spacing from canonical header: `[float(s) for s in canonical.header.get_zooms()[:3]]`
4. Compute auto-windowing: filter non-zero voxels (`data[data != 0]`), compute `np.percentile(nonzero, [5, 95])`, derive `window_center = (p5 + p95) / 2`, `window_width = max(p95 - p5, 1)`
5. Return both data and an extended metadata dict containing: `dimensions` (list of 3 ints from canonical.shape[:3]), `voxel_spacing` (list of 3 floats), `dtype` ("float32"), `window_center` (float), `window_width` (float)

Modify the DICOM loader (`load_dicom_volume` in server/loaders/dicom_loader.py):
1. After assembling the 3D array from DICOM slices, construct a nibabel Nifti1Image using the computed affine from ImageOrientationPatient + ImagePositionPatient + PixelSpacing
2. Apply `nib.as_closest_canonical()` and `np.ascontiguousarray()` — same normalization as NIfTI
3. Compute the same auto-windowing percentiles on the normalized data
4. Return (data, metadata) with same fields as NIfTI loader

Update `VolumeMetadata` in server/catalog/models.py:
1. Add optional fields: `voxel_spacing: list[float] | None = None`, `window_center: float | None = None`, `window_width: float | None = None`

IMPORTANT per D-09: voxel_spacing must be in RAS+ axis order after normalization, not the original file axis order.
